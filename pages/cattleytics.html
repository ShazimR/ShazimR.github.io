CATTLEytics

Capstone case study • ML + backend systems

A full-stack capstone project to predict cattle offspring traits from parental/genetic data and serve predictions through a production-style backend.

This page is intentionally written as a public case study (the repository is private).

Overview
Goal
Predict multiple offspring metrics (multi-target regression/classification style outputs) using sire + dam features and historical records.
Stack
Go backend (REST), Postgres, Python ML microservice (gRPC), and a next.js frontend consuming the API.
My role
Owned the Go backend + DB design, API contracts, and ML serving integration (gRPC).
If you want to talk architecture tradeoffs, deployment, or model-serving decisions, hit “Contact”.
Architecture
  • Go API: authentication + request validation + REST endpoints for predictions and data access.
  • Postgres: structured storage for cattle records, features, and inference-related metadata.
  • Python gRPC: loads trained model(s), exposes strongly-typed inference RPCs, returns predictions to Go.
  • Frontend: calls the Go API; Go fans out to Postgres + gRPC as needed.
What I built
  • Designed API routes and DTOs for model inference requests/responses.
  • Defined gRPC service contracts (protobufs) and implemented client/server integration.
  • Built a clean persistence layer for feature records and related entities.
  • Production-minded error handling, request logging, and validation patterns.
