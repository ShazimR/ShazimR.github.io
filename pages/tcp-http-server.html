Highlights
  • Reads from a TCP connection, parses request line + headers + body.
  • Writes status line + headers + body with correct formatting.
  • Designed with internal packages: request/response/router/server.
  • Test coverage around parsing + response formatting.
  • Explores real-world edge cases: partial reads, CRLF framing, malformed input.
  • Room to expand: chunked encoding, keep-alive, streaming responses, middleware.
  • Fits nicely with your systems/backend interest and interview storytelling.
Tech notes
  • Language: Go
  • IO model: blocking sockets with buffered reads
  • Focus: correctness + clarity > micro-optimizations
