Overview

A full-stack capstone system for predicting cattle offspring traits from parental and historical feature data. Designed as a production-style backend with a separate model-serving microservice.

If you want to talk architecture tradeoffs, deployment, or model-serving decisions, hit “Contact”.
Model: MPCFIN (Implemented From Scratch)

Implemented the Multiplex Cross-Feature Interaction Network (MPCFIN) entirely from scratch in PyTorch by translating the mathematical formulation from a research paper into working code.

  • Custom graph-based feature interaction layers.
  • Manual message-passing logic (no prebuilt architecture).
  • Custom training loop with batching, loss computation, and evaluation.
  • Early stopping and overfitting mitigation on small datasets.
  • Inference service exposed via gRPC.
Backend Architecture
  • Go API: REST endpoints, validation, error handling, orchestration.
  • Postgres: structured feature and metadata storage.
  • Python gRPC microservice: model loading + inference execution.
  • Strict separation between API layer and ML runtime.
