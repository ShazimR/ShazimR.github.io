Overview

A from-scratch HTTP/1.1 server written directly on top of raw TCP sockets in Go. This project intentionally avoids net/http to deeply understand framing, connection lifecycles, parsing state machines, and real-world browser behavior.

Core Capabilities
  • Manual HTTP request parsing with an explicit state machine.
  • Supports both Content-Length and chunked request bodies.
  • Chunked transfer encoding for streaming responses (with trailers).
  • Persistent connections (HTTP/1.0 + HTTP/1.1 semantics).
  • Correct distinction between 404 and 405.
  • Range requests with 206 Partial Content + 416.
  • Binary-safe I/O for video/media streaming.
What This Demonstrates
  • Deep understanding of HTTP over TCP.
  • Ability to design clean, minimal abstractions.
  • Attention to protocol correctness and edge cases.
  • Production-aware thinking (timeouts, shutdown, streaming, containers).
